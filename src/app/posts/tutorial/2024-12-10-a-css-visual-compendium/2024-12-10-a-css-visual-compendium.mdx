What is this, you ask? It's a way for me to review CSS silly. 

No, but actually, it's a tool to see how different CSS properties work in real-time
using an <span className="bg-[#D3D3D3] border-[1px] border-black">interactive element</span> that can be styled with sliders, inputs, etc. (inspired by Mozilla's CSS Reference)

I've always wanted to make something like this so you can really clearly how changing the CSS properties' values affects the element.

There are many CSS frameworks and component libraries out their in the age of React, (for example, I'm using Taliwind and shad/cn for this blog),
but I've found as I work on my projects that I'm lacking in some fundamental knowledge about CSS. So here we are.

<ScrollspyHeader header={'Box Model'} level={3} />
When you give height and width a number, these are in pixels (count em if you don't believe me!).

When I first started learning CSS, I'd always mix up `margin` and `padding`.
Padding is added *inside* the container and margin is *outside*. One way to remember, is if you think of the element as a box (shipping box if you will), you add padding (foam peanuts) on the inside.

<ICMBoxModel />

You can also adjust width and height with other units besides numbers as well as some in other units (em, rem, vh).

Try changing the %'s below. Percent is always relevant to the *parent* element.

<ICMBoxModelPercent />

<ScrollspyHeader header={'Positioning and Display'} level={3} />

Try setting position to `sticky` and scrolling. Pretty useful for things like nav menus and things that you want to stick on the screen. 
Just as % size is relative to the parent, you'll learn that position is also relative to the parent. 

(A common mistake I've made in the past is having an extra div around your sticky element and wondering why the CSS seemed to not work... 
make sure you're applying the style to the right element!)

<ICMPositionDisplay />

For `absolute` and `fixed`, you'll notice the interactive element "leaves" the container to the top of the page and screen, respectively.  
Also, for `absolute`, `fixed`, and `sticky`, you need to specify one of (top, left, right, bottom) so CSS knows where to stick your element (I set top to 5px for demonstration).

<ScrollspyHeader header={'Flexbox'} level={3} />

Now, honestly, flexbox, or `display: flex` has been and still is one of the more confusing topics in css for me.

The general idea is that rather than specifying exact sizes or percentages, you give each item a relative number. And then, the elements kinda... 
work out how spacing goes. Try it out yourself:

<ICMFlexbox />

You'll notice past flex:4 the interactive element stops growing, that's because I've set the <span className='border-[1px] border-red-400'>red container</span> to a fixed size, and there's no space left for the interactive element to grow.

The default for a flex item, (a child of a flexbox div), is flex: 1 (shorthand for flex-grow, flex-shrink, and flex-basis, but I think most of the time you can use the shorthand). 
If all items are 1, they take up equal space, but if you set one item to flex 2, it will (try) take up double it's default space.

Grid is a lot more straightforward to understand imo.
With each child element, you define where it starts (gridColumnStart/gridRowStart), and where it ends (gridColumnEnd/gridRowEnd).
